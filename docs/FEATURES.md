## 1. 핵심 기능 (동작 중)

### 1-1. 사용자 문서 분석
- **URL 분석** (`analyzeUrlDirect`): 웹 페이지 URL 입력 → 텍스트 추출 → 클로드 AI 분석
- **파일 분석** (`analyzeDocumentDirect`): PDF/문서 업로드 → 클로드 AI 분석
- **분석 결과**: 총점 + 15개 카테고리별 점수 + 상세 피드백
  - 기본 5개: 논리력, 구체성, 가독성, 기술이해, 창의성
  - 게임디자인 10개: 게임메카닉스설계, BM설계, 밸런싱, UX/UI, 등
- **AI 엔진**: Anthropic Claude (사용자 분석용)

### 1-2. 합격자 포트폴리오 관리 (관리자)
- PDF/엑셀/CSV 파일 업로드 → Supabase Storage 저장
- **AI 엔진**: Google Gemini 2.0 Flash (합격자 분석용)
- 분석 결과: 점수, 강점, 약점, 요약, 회사, 태그 자동 추출
- 현재 **187개** 합격자 포트폴리오 학습 완료

### 1-3. 결제 시스템
- TossPayments 연동
- 크레딧(회차권) 방식: 1회 분석당 1크레딧 차감
- 결제 페이지, 성공/실패 처리

### 1-4. 인증/계정
- Supabase Auth (소셜 로그인)
- 마이페이지: 분석 이력 조회, 프로젝트 관리
- 관리자 이메일 기반 권한 분리

## 2. 회사별 비교 기능 (⚠️ 문제 있음)

### 현재 동작 방식
1. 187개 합격자 포트폴리오에서 **메타데이터만** 추출 (점수, 태그, 강점/약점 요약)
2. 회사별 평균 점수, 공통 강점/약점 패턴 계산
3. 이 숫자 데이터를 클로드에 전달
4. 클로드가 8개 회사(넥슨, 엔씨소프트, 넷마블, 크래프톤, 스마일게이트, 펄어비스, 네오위즈, 웹젠) 각각에 대해 비교 피드백 생성

### 문제점: "같은 말 반복"
- **원인 1**: 클로드에 전달되는 건 숫자(점수/태그)뿐 — 실제 합격자 포트폴리오 내용(텍스트)이 전달되지 않음
- **원인 2**: 데이터가 부족한 회사(샘플 2개 미만)는 전체 평균값으로 대체 → 여러 회사가 동일한 수치
- **원인 3**: 8개 회사 전부 필수 작성 지시 → 데이터 없는 회사도 억지로 작성 → 뻔한 말 반복
- **결과**: "넥슨 합격자 평균 85점, 귀하는 78점으로 다소 낮습니다" 같은 말이 8개 회사에 반복

### 해결 방향
- **벡터 서치(유사 합격 사례 검색)**: 실제 합격자 포트폴리오 텍스트를 AI에 전달하여 진짜 내용 기반 비교 가능하게
- 데이터 없는 회사는 피드백 생략 또는 "데이터 부족" 표시
- 회사별 고유한 특징을 드러내는 프롬프트 개선

## 3. 유사 합격 사례 검색 (🔧 구현 중)

### 목표
사용자가 포트폴리오를 제출하면, 187개 합격자 중 **가장 비슷한 합격 포트폴리오**의 실제 텍스트를 찾아서 AI에 제공. 이를 통해 "이 부분은 넥슨 합격자 A는 이렇게 썼는데, 당신은 이렇게 써서 차이가 난다" 같은 구체적 피드백 가능.

### 기술 구성
- OpenAI text-embedding-3-small: 텍스트를 벡터(숫자 배열)로 변환
- Supabase pgvector: 벡터 유사도 검색
- portfolio_chunks 테이블: 포트폴리오를 800자 단위 청크로 저장

### 현재 진행 상태
| 단계 | 상태 | 설명 |
|------|------|------|
| DB 테이블 생성 (portfolio_chunks) | ✅ 완료 | SQL 마이그레이션 작성 완료 |
| 임베딩 코드 작성 | ✅ 완료 | lib/vector-search.ts, lib/openai-embedding.ts |
| 분석 함수에 벡터 서치 연동 | ✅ 완료 | analyze.ts에 반영 |
| 관리자 일괄 임베딩 실행 | ❌ 진행 중 | 서버 액션 타임아웃 문제. DB 연결은 성공, OpenAI 호출 단계에서 막힘 |
| 신규 업로드 시 자동 임베딩 | ✅ 코드 완료 | 임베딩 실행이 되면 자동으로 작동 |

### 막힌 부분 상세
- Vercel 서버 액션에서 OpenAI API를 호출하면 "An unexpected response" 에러
- DB 연결 테스트는 성공 (포트폴리오 187개 확인)
- portfolio_chunks 테이블 쿼리 또는 OpenAI 호출 단계에서 실패 추정
- Vercel 대시보드에서 Max Duration 300초로 설정 완료
