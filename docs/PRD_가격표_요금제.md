## 2. 구독 요금제

### 2.1 플랜 비교표

| 플랜 | 가격 | 기간 | 분석 횟수 | 프로젝트 | AI 모델 | 비고 |
|------|------|------|-----------|----------|---------|------|
| **무료 체험** | 0원 | - | 총 1회 | 1개 | Claude Sonnet | 회원가입 즉시 |
| **월 구독** | 17,900원/월 | 1개월 | 무제한 | 무제한 | Claude Sonnet | 인기 플랜 |
| **3개월 패스** | 49,000원 | 3개월 | 무제한 | 무제한 | Claude **Opus** | 프리미엄 AI |
| **컨설팅 포함** | 컨설팅 결제 시 | 1개월 | 무제한 | 무제한 | Claude Sonnet | 자동 부여 |

### 2.2 할인 정책

| 할인 이름 | 대상 플랜 | 정가 | 할인가 | 할인액 | 조건 |
|-----------|-----------|------|--------|--------|------|
| 게임캔버스 수강생 | 월 구독 | 17,900원 | **5,900원** | 12,000원 | 할인코드 입력 |
| 3개월 묶음 | 3개월 패스 | 53,700원 (17,900×3) | **49,000원** | 4,700원 | 자동 적용 |

### 2.3 할인코드 관리
- 환경변수: `GAMECANVAS_DISCOUNT_CODES` (쉼표 구분)
- 검증: `app/actions/payment.ts` → `validateDiscountCode()`
- 적용 대상: **monthly 플랜만** (three_month은 대상 아님)
- 할인 적용 시 주문명: "디자이닛 월 구독 (게임캔버스)"

## 5. 가격 변경 절차

```
1. 이 문서(PRD_가격표_요금제.md) 먼저 수정
2. app/actions/payment.ts의 PLAN_PRICES 수정 (서버 기준값)
3. 위 4.1~4.3 표의 모든 파일에서 UI 가격 수정
4. 테스트: 결제 페이지에서 실제 금액 확인
5. 커밋 & 배포
```

> ⚠️ **주의**: 서버(payment.ts)와 UI 가격이 다르면 결제 오류 발생.
> 컨설팅은 서버에서 DB 금액을 재확인하므로, DB에 이미 생성된 주문의 금액은 수정 불가.

## 6. 가격 표시 규칙

| 규칙 | 예시 |
|------|------|
| 천 단위 쉼표 | 17,900원 (O) / 17900원 (X) |
| 할인 시 원가 취소선 | ~~53,700원~~ → 49,000원 |
| 할인율 뱃지 | "10% 할인" 뱃지 표시 |
| 월/기간 표기 | "원 / 월", "원 / 3개월" |
| 무료는 숫자 | "0원" (O) / "무료" → 별도 문구로 표시 |

## 7. 알려진 이슈

| 항목 | 설명 |
|------|------|
| 가격 중복 관리 | 같은 가격이 5~6곳에 하드코딩 → 공통 상수 파일로 통합 필요 |
| 3개월 원가 불일치 | 53,700원 = 17,900×3인데 UI에만 표시, 서버에서는 49,000원만 사용 |
| 할인코드 UI 위치 | 월 구독 결제 페이지에만 있음 → 3개월에는 할인 적용 불가 |
| 컨설팅 가격 서버 미검증 | tutoring/page.tsx에서 하드코딩한 금액이 그대로 DB에 저장됨 |
