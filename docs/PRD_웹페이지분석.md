# 웹페이지 분석 기능 — 기술 스택 비교 및 로드맵

> 상태: **UI 비활성화** (서버 코드는 유지)
> 최종 갱신: 2026-03-26

---

## 현재 상태

- `analyzeUrlDirect()` 함수는 `app/actions/analyze.ts`에 구현 완료
- UI 탭(URL 링크)은 비활성화 상태 — 서비스 안정성을 위해 숨김 처리
- 서버 코드는 삭제하지 않고 유지 (향후 재활용)

### 현재 구현된 크롤링 방식

```
사용자 URL 입력
    ↓
1차: 직접 fetch (30초 타임아웃)
    ↓ (텍스트 500자 미만이면)
2차: Jina AI Reader 폴백 (r.jina.ai/{url})
    ↓
HTML → 텍스트 추출 (script/style/nav 제거)
    ↓
Claude AI 텍스트 분석 (이미지 분석 불가)
```

## 기술 옵션 비교

### 옵션 A: 현재 방식 (fetch + Jina AI Reader)

**작동 원리**: 서버에서 URL을 직접 fetch하여 HTML을 텍스트로 변환. SPA는 Jina AI가 렌더링 대행.

| 항목 | 내용 |
|------|------|
| 장점 | 이미 구현 완료. 추가 비용 없음 (Jina 무료 티어) |
| 단점 | SPA 일부 제한. 이미지/레이아웃 분석 불가. 노션 차단 가능 |
| 비용 | **무료** |
| 분석 품질 | 텍스트만 분석 — 게임디자인 역량 15개 항목 평가 가능. 가독성/레이아웃 평가는 추정치 |
| 구현 난이도 | ✅ 완료 |

### 옵션 B: Puppeteer / Playwright (헤드리스 브라우저)

**작동 원리**: 서버에서 실제 브라우저를 띄워 페이지를 렌더링. 스크린샷 캡처 + 텍스트 추출 모두 가능.

| 항목 | 내용 |
|------|------|
| 장점 | SPA 완벽 지원. 스크린샷으로 레이아웃/가독성 분석 가능. 노션 등 JS 렌더링 페이지 대응 |
| 단점 | **Vercel에서 직접 실행 불가** — 별도 서버(AWS Lambda, Railway 등) 필요. 느림 (5-15초) |
| 비용 | 서버 비용 **월 $5-20** (트래픽에 따라) |
| 분석 품질 | 텍스트 + 스크린샷 → Claude에 이미지로 전달 → PDF 수준의 분석 품질 |
| 구현 난이도 | 중간 (별도 서버 구축 + API 연동) |

### 옵션 C: 외부 렌더링 API (Browserless / ScrapingBee / ScreenshotOne)

**작동 원리**: 외부 서비스가 브라우저 렌더링 + 스크린샷 캡처를 대행. Vercel에서 API 호출만 하면 됨.

| 항목 | 내용 |
|------|------|
| 장점 | Vercel 호환 (서버 불필요). SPA 완벽 지원. 스크린샷 제공 |
| 단점 | 외부 서비스 의존. API 비용 발생 |
| 비용 | 건당 **$0.001-0.01** (월 1000건 = $1-10) |
| 분석 품질 | 텍스트 + 스크린샷 → PDF 수준 분석 |
| 구현 난이도 | 쉬움 (API 호출 + 결과를 Claude에 전달) |

**대표 서비스:**
- [Browserless](https://browserless.io) — 월 $10부터, 스크린샷 + PDF 변환
- [ScrapingBee](https://scrapingbee.com) — 월 1000건 무료, 스크린샷 지원
- [ScreenshotOne](https://screenshotone.com) — 월 100건 무료, 스크린샷 전문

## 권장 로드맵

### Phase 1: 현재 (완료)
- UI 탭 숨기기
- PDF 분석에 집중

### Phase 2: 단기 (계약 안정화 후)
- 현재 fetch+Jina 방식 품질 테스트
- 텍스트 기반 분석만으로도 충분한지 평가
- 충분하면 탭 재오픈

### Phase 3: 중기 (사용자 요청 시)
- 외부 API(옵션 C) 도입
- 스크린샷 캡처 → Claude에 이미지로 전달
- 가독성/레이아웃 분석 품질 PDF 수준으로 향상

## 기술 결정 시 고려사항

1. **사용자가 웹페이지 분석을 얼마나 요청하는가?** — 수요 없으면 투자 불필요
2. **텍스트만으로 충분한가?** — 게임 기획서는 텍스트 비중이 높아 A 방식으로도 80% 커버 가능
3. **노션 포트폴리오 비중** — 노션은 SPA라 fetch로 안 되고 Jina 의존. 비중이 높으면 옵션 C 추천
4. **예산** — 옵션 C가 가성비 최적 (무료 티어로 시작 가능)
