### 2-1. 환불 규정에 회차권(크레딧) 추가

**상태**: 미착수
**파일**: `app/refund-policy/page.tsx`

#### 현재 상태
- 환불 규정에 **구독(월/3개월)만** 명시됨
- 회차권(크레딧) 환불 규정이 **완전히 누락**

#### 변경 내용

1. **제1조 (목적)** — "회차권(크레딧) 결제" 문구 추가
2. **제2조 (서비스 설명)** — 회차권 패키지 목록 추가
   - 1회권: 2,900원
   - 5회권: 7,900원 (회당 1,580원)
   - 10회권: 12,900원 (회당 1,290원)
3. **제5조 (회차권 환불 기준)** — 신규 조항 추가
   - 미사용 전체 회차: 결제일 7일 이내 전액 환불
   - 부분 사용: 사용 회차 × 정가(2,900원) 차감 후 잔여 금액 환불
     - 예시: 5회권(7,900원) 구매 후 2회 사용 → 7,900 - (2,900 × 2) = 2,100원 환불
   - 환불 불가: 전부 사용 또는 결제일 7일 경과
4. **환불 요약 표** — 회차권 행 추가

### 2-2. 마이페이지 게이지 UI

**상태**: 미착수
**파일**: `app/mypage/page.tsx`

#### 현재 상태
- 단순 텍스트로 "무제한" 또는 "N회" 표시
- 시각적 직관성 부족

#### 변경 내용

**크레딧 사용자** (무료 또는 만료된 구독):
```
남은 크레딧
[======----] 7회 남음
```
- 프로그레스 바 (최대값 10 기준, 초과 시 풀 게이지)
- 색상: `bg-[#5B8DEF]` (브랜드 파란색)

**구독자** (활성 유료 구독):
```
구독 기간
[========--] D-15
2026.03.06 ~ 2026.04.06
```
- started_at ~ expires_at 기간 중 경과 비율로 게이지
- 남은 일수(D-day) 표시
- 색상: `bg-emerald-400` (초록)

**크레딧 + 구독 동시 보유** (2-3 정책과 연동):
- 크레딧 게이지 먼저 표시
- 그 아래 구독 게이지 표시
- 안내: "보유 회차(N회)를 먼저 소모한 뒤 구독이 적용됩니다"

#### 기술 구현
- `Subscription` 타입에 `analysis_credits`, `started_at` 필드 추가
- shadcn/ui `Progress` 컴포넌트 활용

### 2-3. 크레딧 우선 소모 로직 (핵심 정책 변경)

**상태**: 미착수
**파일**: `app/actions/subscription.ts`

#### 현재 정책
> 구독이 활성이면 무제한 (크레딧 무시) → 구독 없으면 크레딧 차감

#### 변경 정책
> **크레딧이 있으면 무조건 크레딧 먼저 차감 → 크레딧 0이면 구독 사용**

이 정책의 의미: 회차권을 산 뒤 구독을 하면, 회차부터 전부 쓰고 나서 구독이 적용됨.

#### 함수별 변경

**`deductCredit()` (225행)**
```
기존: ① 활성 구독 → 차감 안 함    ② 그 외 → 크레딧 -1
변경: ① 크레딧 > 0 → 크레딧 -1    ② 활성 구독 → 차감 안 함    ③ 그 외 → 에러
```

**`checkAnalysisAllowance()` (185행)**
```
기존: ① 활성 구독 → 무제한        ② 크레딧 확인
변경: ① 크레딧 > 0 → 허용(남은 수)  ② 활성 구독 → 무제한    ③ 그 외 → 차단
```

#### 구독 결제 시 크레딧 처리
- `processSubscriptionPayment()` 변경 불필요
- 구독은 즉시 시작, 크레딧은 그대로 유지
- 분석할 때마다 크레딧부터 소모 → 0이 되면 자연스럽게 구독으로 전환
