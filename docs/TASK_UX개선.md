# UX 개선 태스크

> 마지막 업데이트: 2026-03-07
> 관련 페이지: 분석 페이지(`app/analyze/`), 마이페이지(`app/mypage/`)

---

## 1. 분석 페이지 (`/analyze`)

### 1-1. 프로젝트 선택 → 문서 업로드 흐름 개선
- **현재**: "프로젝트 선택" 섹션과 "문서 분석" 섹션이 별도 카드로 분리
- **문제**: 프로젝트를 먼저 선택해야 하는지, 문서를 먼저 올려도 되는지 흐름이 불명확. 두 섹션 사이에 관계성이 시각적으로 드러나지 않음
- **개선안**:
  - [ ] 프로젝트 미선택 시 업로드 영역에 "프로젝트를 먼저 선택해주세요" 안내 표시
  - [ ] 또는 스텝 인디케이터(① 프로젝트 선택 → ② 문서 업로드 → ③ 분석) 도입

### 1-2. "새 프로젝트 만들기" 가시성 부족
- **현재**: 점선 테두리에 작은 "+" 아이콘과 텍스트. 프로젝트 목록 하단에 위치
- **문제**: 처음 사용자가 프로젝트 생성 버튼을 발견하기 어려움
- **개선안**:
  - [ ] 프로젝트가 0개일 때 — 큰 CTA 버튼으로 "첫 프로젝트를 만들어보세요"
  - [ ] 프로젝트가 있을 때 — 현재 디자인 유지하되 호버 시 배경색 변화 등 인터랙션 강화

### 1-3. 파일 크기 안내 섹션이 과도한 공간 차지
- **현재**: "더 정확한 분석을 받고 싶나요?" 섹션이 하단에 크게 펼쳐져 있음 (30MB/30~100MB/100MB+ 3단 안내)
- **문제**: 매번 보기엔 정보량이 많고, 핵심 행동(업로드)에서 시선이 분산됨
- **개선안**:
  - [ ] 접이식(Accordion/Collapse)으로 변경 — 기본 접힘, 클릭 시 펼침
  - [ ] 또는 업로드 영역 안에 한 줄 요약만 표시: "권장 30MB 이하 · 자세히 보기"

### 1-4. 업로드 드롭존 시각적 강화
- **현재**: 점선 테두리 안에 작은 업로드 아이콘 + 텍스트
- **문제**: 드래그 앤 드롭 영역이라는 것이 직관적이지 않음. 클릭 유도도 약함
- **개선안**:
  - [ ] 드롭존 높이를 더 키우거나, 호버 시 배경색/테두리색 강조
  - [ ] "파일을 여기에 끌어다 놓으세요" 텍스트를 더 크게
  - [ ] 파일 드래그 중(dragover) 시 영역 전체가 하이라이트되는 애니메이션

### 1-5. 파일 업로드 / URL 링크 탭 구분 강화
- **현재**: 두 탭이 나란히 있지만 선택된 탭(파일 업로드)과 비선택 탭(URL 링크)의 시각 차이가 약함
- **개선안**:
  - [ ] 선택된 탭에 하단 보더 또는 배경색 차이를 더 뚜렷하게
  - [ ] 비선택 탭을 회색 톤으로 더 낮춰서 구분

## 2. 마이페이지 (`/mypage`)

### 2-1. 구독 상태 표시 혼란
- **현재**: "현재 플랜: 월 무제한" + "해지됨" 빨간 뱃지 + "해지일: 2026년 2월 22일" 빨간색
- **문제**:
  - "월 무제한"이 현재 플랜이라고 표시되지만 "해지됨"이면 혼란 (아직 쓸 수 있는 건지?)
  - 해지일이 이미 과거(2026-02-22)인데 빨간색만으로는 "만료됨"인지 "곧 만료"인지 구분 안 됨
  - 남은 크레딧 7회의 게이지가 전체 대비 몇 회인지 표시 안 됨
- **개선안**:
  - [ ] 해지일이 과거면 "만료됨"으로 상태 변경 표시, 갱신/재구독 CTA 버튼 추가
  - [ ] 크레딧 게이지에 "7 / 10" 같은 전체 대비 잔여 표시
  - [ ] 구독 만료 + 크레딧 잔여 상태를 한눈에 이해할 수 있는 문구 추가 (예: "구독은 만료되었지만 남은 크레딧 7회로 분석 가능합니다")

### 2-2. 데이터 안전 안내 배너 반복 노출
- **현재**: "당신의 데이터는 안전합니다" 초록색 배너가 매번 상단에 크게 표시
- **문제**: 반복 방문자에겐 불필요한 공간 차지. 첫 방문에는 유용하지만 매번 보면 노이즈
- **개선안**:
  - [ ] 닫기(X) 버튼 추가 — 닫으면 localStorage에 저장, 이후 미노출
  - [ ] 또는 프로필 섹션 하단에 한 줄 텍스트로 축소

### 2-3. 환불 버튼이 너무 눈에 띔 + 크레딧 내역 정보 밀도
- **현재**: 빨간 테두리의 "환불하기" 버튼이 크고 대놓고 보임. "환불 가능: 4,200원 (3회 사용)"도 바로 옆에 표시
- **문제**:
  - 환불 버튼이 지나치게 강조되어 환불을 유도하는 느낌을 줌
  - 서비스 입장에서 환불을 적극 종용하는 인상
  - 정보도 한 줄에 빽빽함
- **개선안**:
  - [ ] "환불하기" 버튼을 텍스트 링크로 축소 (빨간 버튼 → 작은 회색 텍스트 링크)
  - [ ] "환불 가능: N원" 문구를 클릭/호버 시에만 노출하거나, 환불 페이지 진입 후 표시
  - [ ] 환불정책 안내 옆에 작은 "환불 요청" 링크 정도로 충분
  - [ ] 레이아웃: 2행으로 분리 (1행: 구매 정보, 2행: 사용 현황 + 환불 링크)

### 2-4. 프로젝트 인벤토리 등급 필터 과잉
- **현재**: S 90+ / A 80-89 / B 70-79 / C 60-69 / D ~59 필터 버튼 5개가 나열
- **문제**: 프로젝트가 2개뿐인데 필터 UI가 과함. 프로젝트 수가 적을 때 의미 없음
- **개선안**:
  - [ ] 프로젝트 5개 미만이면 필터 숨김
  - [ ] 또는 등급 필터를 드롭다운으로 축소

### 2-5. 프로젝트 카드 정보 및 인터랙션
- **현재**: 폴더 아이콘 + 프로젝트명 + "N개 문서" + "최고 78" + "열기 >"
- **문제**: "열기 >" 링크가 카드 좌하단에 작게 위치, 카드 전체 클릭 불가해 보임
- **개선안**:
  - [ ] 카드 전체를 클릭 가능하게 (cursor: pointer + 호버 효과)
  - [ ] "열기 >" 텍스트 제거, 카드 자체가 링크임을 시각적으로 표현
  - [ ] 마지막 분석일, 평균 점수 등 추가 정보 고려

### 2-6. 페이지 전체 길이 (스크롤 과다)
- **현재**: 5개 섹션(안내배너/프로필/구독/크레딧구매/프로젝트)이 세로로 나열
- **문제**: 스크롤이 길어 하단의 프로젝트 인벤토리까지 도달하기 어려움
- **개선안**:
  - [ ] 프로필 + 구독 상태를 한 카드로 병합 (2컬럼 또는 요약 카드)
  - [ ] 안내 배너 축소/제거 (2-2 참고)
  - [ ] 크레딧 구매 내역을 접이식으로 변경
